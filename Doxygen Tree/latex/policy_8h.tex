\doxysubsection{Detailed Description}
Öffentliches Interface für alle Deadlock-\/\+Behandlungsstrategien. 

Eine {\ttfamily \doxylink{struct_policy}{Policy}} kapselt eine konkrete Strategie zur Vermeidung oder Erkennung von Deadlocks. Das Interface basiert auf Funktionszeigern und ermöglicht so einen austauschbaren Strategy-\/\+Pattern-\/\+Ansatz:\+ Der \doxylink{struct_scheduler}{Scheduler} ruft stets dieselben Callbacks auf, unabhängig davon, welche konkrete \doxylink{struct_policy}{Policy} gerade aktiv ist.

Aktuell werden drei Strategien unterstützt:\+
\begin{DoxyItemize}
\item {\bfseries{Graph-\/\+Erkennung}}:\+ Baut den Wait-\/\+For-\/\+Graphen auf und prüft auf Zyklen.
\item {\bfseries{Bankier-\/\+Algorithmus}}:\+ Genehmigt Anfragen nur bei nachgewiesener Sicherheit.
\item {\bfseries{Hold-\/and-\/\+Wait-\/\+Eliminierung}}:\+ Lehnt Anfragen von Prozessen ab, die bereits Ressourcen halten. 
\end{DoxyItemize}
