\doxysubsection{Detailed Description}
Implementierung des GTK-\/3-\/\+Frontends für den Deadlock-\/\+Simulator. 

Diese Datei implementiert die gesamte grafische Benutzeroberfläche. Das Fenster besteht aus folgenden Bereichen:\+
\begin{DoxyItemize}
\item {\bfseries{Header}}:\+ Titel und Untertitel der Anwendung.
\item {\bfseries{Button-\/\+Leiste}}:\+ Drei Policy-\/\+Buttons und ein Step-\/\+Button.
\item {\bfseries{Statusleiste}}:\+ Aktive \doxylink{struct_policy}{Policy}, Verschiebungs-\/\+Zähler, aktueller Tick.
\item {\bfseries{Hauptbereich}}:\+ Scrollbares Textfenster mit Systemzustand-\/\+Snapshot (links) und statische Legende (rechts).
\end{DoxyItemize}

Die Simulation arbeitet mit einem prozeduralen Prozessmodell:\+ Jeder Prozess durchläuft die Zustände IDLE → RUNNING → DONE und wird dann neu gestartet. Alle Ressourcenanforderungen und -\/freigaben werden als Ereignisse in den \doxylink{struct_scheduler}{Scheduler} eingespeist.
