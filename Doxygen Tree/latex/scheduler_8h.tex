\doxysubsection{Detailed Description}
Öffentliches Interface für den ereignisgesteuerten Simulator-\/\+Kern. 

Der {\ttfamily \doxylink{struct_scheduler}{Scheduler}} ist das zentrale Steuerungselement der Simulation. Er verwaltet eine zeitgeordnete Event-\/\+Queue (Min-\/\+Heap), delegiert Ressourcenanforderungen an die aktive {\ttfamily \doxylink{struct_policy}{Policy}} und aktualisiert den {\ttfamily \doxylink{struct_system_state}{System\+State}} entsprechend dem Ergebnis.

Die interne Struktur ist nach dem Opaque-\/\+Pointer-\/\+Muster versteckt (Information Hiding):\+ Externe Module können nur über die hier deklarierten Funktionen mit dem \doxylink{struct_scheduler}{Scheduler} interagieren.

Verarbeitet alle Ereignisse bis einschließlich {\ttfamily until\+\_\+time}. 

In einer Schleife werden Ereignisse aus dem Heap entnommen und typ-\/abhängig verarbeitet:\+
\begin{DoxyItemize}
\item {\ttfamily EV\+\_\+\+REQUEST:\+} Delegiert an die \doxylink{struct_policy}{Policy}; bei Ablehnung wird das Ereignis mit erhöhtem {\ttfamily retries-\/\+Zähler} erneut eingeplant (max. {\ttfamily MAX\+\_\+\+RETRIES} mal).
\item {\ttfamily EV\+\_\+\+RELEASE:\+} Gibt die angegebene Menge einer Ressourcenklasse frei.
\item {\ttfamily EV\+\_\+\+TERMINATE:\+} Gibt alle vom Prozess gehaltenen Ressourcen frei.
\item {\ttfamily EV\+\_\+\+CHECKPOINT:\+} Platzhalter, aktuell ohne Aktion.
\end{DoxyItemize}

Nach jedem verarbeiteten Ereignis wird der {\ttfamily on\+\_\+tick-\/\+Callback} der \doxylink{struct_policy}{Policy} aufgerufen, falls vorhanden.
