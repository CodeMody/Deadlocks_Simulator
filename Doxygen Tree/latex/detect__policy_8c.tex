\doxysubsection{Detailed Description}
Implementierung der Wait-\/\+For-\/\+Graph-\/\+Zykluserkennungs-\/\+Policy. 

Diese \doxylink{struct_policy}{Policy} erkennt potenzielle Deadlocks, indem sie nach jeder Ressourcenanforderung den Wait-\/\+For-\/\+Graphen aufbaut und mittels iterativer Tiefensuche (DFS) auf Zyklen prüft. Ein Zyklus bedeutet, dass eine zirkuläre Warteabhängigkeit entstehen würde — eine der notwendigen Bedingungen für einen Deadlock.

Prozess A wartet auf Prozess B ({\ttfamily adj}\+[A]\+\+[B]\+ = true), wenn:\+
\begin{DoxyItemize}
\item A mindestens eine Ressource noch benötigt (request\+[A]\+\+[r]\+ \texorpdfstring{$>$}{>} 0) und
\item B genau diese Ressourcenklasse aktuell hält (allocation\+[B]\+\+[r]\+ \texorpdfstring{$>$}{>} 0).
\end{DoxyItemize}

Die Anfrage wird nur genehmigt, wenn durch die simulierte Zuweisung kein Zyklus im Wait-\/\+For-\/\+Graphen entsteht.
