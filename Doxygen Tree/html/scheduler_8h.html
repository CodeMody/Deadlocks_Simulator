Detailed Description

Öffentliches Interface für den ereignisgesteuerten Simulator-Kern.

Der Scheduler ist das zentrale Steuerungselement der Simulation. Er verwaltet eine zeitgeordnete Event-Queue (Min-Heap), delegiert Ressourcenanforderungen an die aktive Policy und aktualisiert den SystemState entsprechend dem Ergebnis.

Die interne Struktur ist nach dem Opaque-Pointer-Muster versteckt (Information Hiding): Externe Module können nur über die hier deklarierten Funktionen mit dem Scheduler interagieren.

◆ scheduler_run_until()

void scheduler_run_until ( Scheduler * s,
uint64_t until_time )

Verarbeitet alle Ereignisse bis einschließlich until_time.

In einer Schleife werden Ereignisse aus dem Heap entnommen und typ-abhängig verarbeitet:

  • EV_REQUEST: Delegiert an die Policy; bei Ablehnung wird das Ereignis mit erhöhtem retries-Zähler erneut eingeplant (max. MAX_RETRIES mal).
  • EV_RELEASE: Gibt die angegebene Menge einer Ressourcenklasse frei.
  • EV_TERMINATE: Gibt alle vom Prozess gehaltenen Ressourcen frei.
  • EV_CHECKPOINT: Platzhalter, aktuell ohne Aktion.

Nach jedem verarbeiteten Ereignis wird der on_tick-Callback der Policy aufgerufen, falls vorhanden.

Parameters
schZeiger auf den Scheduler.
until_timeLogischer Zeitstempel, bis zu dem Events verarbeitet werden.

Verarbeitet alle Ereignisse bis einschließlich until_time.

Die Hauptschleife der Simulation: Solange der Heap nicht leer ist und das nächste Ereignis im Zeitfenster liegt, wird es entnommen und typ-abhängig verarbeitet. Nach jedem Ereignis wird der on_tick-Callback der Policy aufgerufen, sofern vorhanden.

Parameters
